# Adding a New Product — Instructions for the AI Assistant

Read this whenever the owner says something like "add this new PDF to the site" or gives you a product to add. This is the complete, repeatable process for keeping `src/constants.ts`, `PDF_CATALOG.md`, and what's actually uploaded to R2 all in sync.

## The owner's side (already done before they talk to you)

The owner will have already:
1. Picked a product id in `lowercase-with-hyphens` format (the only format the admin panel accepts — see `admin/netlify/functions/lib/validation.ts`, pattern `/^[a-z0-9][a-z0-9-]*$/`).
2. Uploaded the PDF to R2 via the Admin Panel's Files tab, using that exact id.
3. Told you the id, title, price, description, category, and any other details (language, level, whether it should appear on the storefront yet, etc.).

**The id they give you is final and must not be changed.** It has to match the R2 filename exactly (case-sensitive), or the whole thing breaks — the id is what get-pdf.ts and get-my-library.ts use to look up the actual file and check buyer access.

## Your job: add it to `src/constants.ts`

### Step 1 — Check it doesn't already exist

Search `src/constants.ts` for the exact id string first. **Never add an id that's already present** — `PRODUCTS` is a flat array, and if two entries share an id, "My Library" renders the PDF twice (once correctly, once with whatever the duplicate says). This exact bug happened before (three duplicate `pl-*` entries existed alongside their auto-generated equivalents) and was the whole reason this file exists — don't reintroduce it.

This also matters if the id fits an already-generated pattern (e.g. `pl-python-beginner-en` is auto-generated by the `PROGRAMMING_LANGUAGES` × `PROGRAMMING_LEVELS` × language matrix near the top of the file) — if it's already covered by a generator, you don't need to add anything at all. Just confirm the generated entry's `available` flag is `true` if the owner wants it live.

### Step 2 — Decide: does it extend an existing generator, or is it a one-off?

- **Extends an existing pattern** (new programming language, new tool/AI item in an existing family): add it to the relevant seed array near the top of `constants.ts` (`PROGRAMMING_LANGUAGES`, `WEB_DEVELOPMENT_ITEMS`, `TOOLS_ITEMS`, `AI_ITEMS`, `AI_AUTOMATION_ITEMS`, `PRODUCTIVITY_ITEMS`, etc.), following the exact shape of the neighboring entries in that array. The generator will produce the full `Product` object(s) automatically — do not also hand-write a duplicate entry for it.
- **One-off / doesn't fit any pattern**: add a plain object directly to the `PRODUCTS` array, in the manually-added block near the end (next to `bsit-complete`, `bscs-complete`, `ai-claude-notes-en`, etc.). Use this exact shape:

```ts
{
  id: 'exact-id-the-owner-gave-you',
  title: 'Whatever they told you',
  description: 'Whatever they told you',
  price: 0,
  thumbnail: PREORDER_THUMBNAIL, // see Step 3
  mobileUrl: MOBILE_URL,
  desktopUrl: DESKTOP_URL,
  category: 'Whatever they told you', // see Step 4
  available: true
}
```

### Step 3 — Thumbnail

- If the owner gives you an actual thumbnail image/path, use it.
- If not, use `PREORDER_THUMBNAIL` (the existing generic placeholder constant) rather than inventing a path to an image that doesn't exist — a broken image path is worse than a generic one.
- **Never guess at reusing another product's real thumbnail** just to avoid a blank one — that would be visually wrong for two different products.

### Step 4 — Category

- If it belongs in an existing category (`Programming Languages`, `Web Dev & Tools`, `Frameworks`, `Claude Notes`, `AI Tools & Automations`, `ADVANCED STUDY`), use that exact string — it must match a value already in the `CATEGORIES` array for the product to show up in that storefront section.
- If the owner wants it granted to specific buyers only, not shown on the public storefront (like `bsit-complete`/`bscs-complete`), use `category: 'Unlisted'` and `available: true` — `Unlisted` is intentionally not in `CATEGORIES`, so it never renders on the homepage but is still a fully resolvable product for `/view/<id>` and "My Library".
- If it's a brand new category, ask the owner to confirm before adding it to `CATEGORIES` — that changes the storefront's navigation, which is a visible site-wide change worth a quick confirmation.

### Step 5 — Price, language, level

- `price`: exactly what the owner told you (in pesos, matching the existing entries' currency convention — no symbol, just the number).
- `language`: only set if this product should be filterable by English/Tagalog in "My Library" (`'en'` or `'tl'`) — omit entirely if not applicable.
- `level`: only relevant for programming-language products (`'beginner' | 'intermediate' | 'advanced' | 'build-phase' | 'activities' | 'package'`) — omit for everything else.

### Step 6 — Verify

Run these two, in order, and fix anything that fails before telling the owner it's done:

```bash
npx tsc --noEmit -p tsconfig.json
npm run build
```

### Step 7 — Regenerate the catalog doc

```bash
npm run catalog:sync
```

This regenerates `PDF_CATALOG.md` directly from `PRODUCTS` — never hand-edit that file, it's overwritten every time this runs. Confirm the new product appears in the regenerated table with the exact id, title, and price the owner asked for.

### Step 8 — Tell the owner what to check

After all of the above, tell the owner:
- The exact product id you used (so they can double check it against what they typed into the Admin Panel upload).
- To open the Admin Panel's Files tab and confirm a file with that exact id already exists (if they haven't uploaded it yet, the entry will exist in the catalog but 404 when opened until they do).
- That granting it to a buyer (Admin Panel → Buyers) will now show the real title/thumbnail/price in "My Library" immediately — no further catalog step needed for that buyer.

## What NOT to do

- Don't touch anything in `admin/` for this task — the admin panel's job is uploading files and granting access; it has no product-catalog UI by design, and that's intentional (the owner explicitly wants the catalog to stay hand-managed in code, not become a CMS).
- Don't remove or "clean up" other unrelated entries in `constants.ts` while you're in there, even if something looks odd — stick to the one product you were asked to add.
- Don't invent a description, price, or thumbnail the owner didn't give you — ask instead of guessing if something's missing.
- Don't skip Step 6 (typecheck + build) — a broken `constants.ts` breaks the entire storefront, not just the new product.
